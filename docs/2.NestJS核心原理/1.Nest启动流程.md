### 一、 环境准备与工程创建

在实际开发中，我们统一使用 NestJS 官方提供的 CLI 工具，它可以确保项目结构的标准化。

**1. 安装 CLI 命令行工具**
```bash
npm install -g @nestjs/cli
```

**2. 初始化项目**
```bash
nest new project-name
```
*   **开发选择**：建议选择 `pnpm` 或 `npm`。在企业级架构中，`pnpm` 因其硬链接机制能极大节省磁盘空间并加快安装速度。

---

### 二、 标准目录结构解析（核心文件）

NestJS 的工程化程度极高，初学者必须理解生成的每一个关键文件的职责：

1.  **`main.ts` (入口文件)**：使用 `NestFactory` 创建应用实例，是整个程序的启动引擎。
2.  **`app.module.ts` (根模块)**：整个应用的“地图”，负责组织和链接所有的 Controller 和 Provider。
3.  **`app.controller.ts`**：处理 HTTP 请求，属于路由层。
4.  **`app.service.ts`**：处理具体业务逻辑，属于服务层（Provider）。
5.  **`nest-cli.json`**：项目编译与打包的配置文件。

---

### 三、 启动与开发模式

**1. 常用命令**
*   `npm run start`: 普通启动（不常用）。
*   `npm run start:dev`: **热重载启动（最常用）**。底层使用的是 `tsc --watch`，监听文件变化并自动重新编译。
*   `npm run build`: 生产环境打包。将 TS 代码编译为标准 JS 存放在 `dist` 目录。

**2. 核心依赖说明**
*   `@nestjs/core`: Nest 核心运行引擎。
*   `@nestjs/common`: 包含大量装饰器、接口和工具类（如 `@Controller`, `@Injectable`）。
*   `reflect-metadata`: 实现装饰器反射的核心库，DI（依赖注入）的基石。

---

### 四、 实际开发建议

1.  **Strict Mode**: 确保 `tsconfig.json` 中的 `strict: true` 开启。NestJS 深度依赖类型系统，关闭严格模式会丧失 TS 的大部分优势。
2.  **平台选型**: NestJS 默认底层使用 **Express**。如果追求极致吞吐量，可以在创建项目后将其切换为 **Fastify**。

---

### 五、 面试题储备

**Q1：NestJS 相比于原生 Express/Fastify 的核心价值是什么？**
*   **A**: 
    1.  **架构约束**：Express 过于灵活，导致大型项目代码凌乱。NestJS 强制使用模块化和分层架构，解决了 Node.js 项目的可维护性痛点。
    2.  **高度解耦**：内置 IoC 容器，通过 DI（依赖注入）实现代码解耦，便于单元测试。
    3.  **开箱即用**：集成了异常过滤器、守卫、拦截器等企业级切面（AOP）功能。

**Q2：`main.ts` 中的 `NestFactory.create(AppModule)` 背后发生了什么？**
*   **A**: 这是项目的初始化起点。它会：
    1.  初始化 **IoC 容器**。
    2.  递归扫描 `AppModule` 及其引入的其他模块。
    3.  解析模块间的依赖关系，并实例化所有相关的 Provider。
    4.  绑定 Controller 路由到具体的底层 Web 平台（如 Express）。
